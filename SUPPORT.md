# Support

Thanks for using groot! Here's how to get help — picking the right channel gets you an answer fastest.

| I want to…                                        | Go to |
| ------------------------------------------------- | ----- |
| Ask a question or discuss an idea                  | [GitHub Discussions](https://github.com/bloxy-studios/groot/discussions) |
| Report a bug                                       | [Bug report form](https://github.com/bloxy-studios/groot/issues/new?template=bug_report.yml) |
| Request a feature                                  | [Feature request form](https://github.com/bloxy-studios/groot/issues/new?template=feature_request.yml) |
| Ask for a new framework/scaffold to be supported   | [Scaffold request form](https://github.com/bloxy-studios/groot/issues/new?template=scaffold_request.yml) |
| Report a security vulnerability                    | [Security policy](./SECURITY.md) — **never** a public issue |
| Understand how groot works                         | [docs/architecture.md](./docs/architecture.md) and [docs/cli-spec.md](./docs/cli-spec.md) |
| Contribute                                         | [CONTRIBUTING.md](./CONTRIBUTING.md) |

## Before opening an issue

1. **Search existing issues** — your problem may already be tracked.
2. **Reproduce on the latest release** — run `bunx create-groot@latest --version` to check yours.
3. **Collect the details** the issue form asks for (OS, architecture, Bun version, install channel, logs). Complete reports get fixed much faster.

## Response expectations

groot is maintained by a small team. We triage new issues regularly, but there is no guaranteed response SLA — being kind and providing complete information is the best accelerator. If an issue is urgent because it's a security problem, use the [security policy](./SECURITY.md) instead.
